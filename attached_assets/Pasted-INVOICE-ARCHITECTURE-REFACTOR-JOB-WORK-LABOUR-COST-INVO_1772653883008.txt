INVOICE ARCHITECTURE REFACTOR + JOB WORK LABOUR COST INVOICE SYSTEM

Refactor the invoice architecture across the application to remove duplication, centralize logic, and extend the same invoice framework to the Job Work → Labour Cost module.

The system must use a shared invoice engine while allowing different modules (Purchase and Job Work) to consume the same components.

1. REMOVE DUPLICATE INVOICE SECTION FROM HOME

Delete the entire Invoices block currently displayed at the bottom of the Home / Dashboard page.

Reason:
Invoice management already exists under the Purchase module. The dashboard must not contain a second invoice interface.

Required action:

Remove UI component

Remove related queries

Remove redundant dashboard widgets referencing invoices

Dashboard should remain a high-level summary only, not an invoice management interface.

2. CENTRALIZE INVOICES UNDER PURCHASE MODULE

Invoices must exist only inside the Purchase module.

Structure:

Purchase
└── Invoices
  ├── All
  ├── Pending
  └── Needs Review

Tab behavior:

All
Shows every purchase invoice.

Pending
Invoices approved but not yet paid.

Needs Review
Invoices flagged due to validation issues.

Filtering, queries, and status logic must be handled inside this module.

3. EXTEND SAME INVOICE SYSTEM TO JOB WORK → LABOUR COST

Take the entire invoice logic and architecture used in the Purchase → Invoices system and implement it inside:

Job Work
└── Labour Cost

Structure:

Labour Cost
├── All
├── Pending
└── Needs Review

The backend logic must remain identical.

Reuse existing systems:

OCR invoice extraction

Vendor detection

Invoice parsing

GST extraction

Validation checks

Status tagging

Review flagging

File attachment storage

Manual override edits

This must use the same invoice engine, not a separate implementation.

4. LABOUR COST SUMMARY METRICS

Add summary cards at the top of the Job Work → Labour Cost page.

Replace existing metrics with:

Total Labour Charge Payable

Definition:
Sum of all labour invoices approved and pending payment.

Upcoming Labour Charges

Definition:
Labour invoices recorded but not yet due.

Discrepancies

Definition:
Invoices flagged due to validation issues including:

Rate per kg mismatch

GST mismatch

Vendor mismatch

OCR extraction uncertainty

Manual review flag

These must appear as summary cards above the invoice list.
