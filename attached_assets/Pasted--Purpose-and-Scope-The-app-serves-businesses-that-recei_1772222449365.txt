. Purpose and Scope The app serves businesses that receive multiple purchase invoices and need: • Clean data extraction from invoices • Party-wise and date-wise visibility • Historical intelligence per vendor • Payment execution and reminders • Traceability between purchases and sales orders (1-to-many and many-to-many) The app is not a document viewer. It is a structured financial system built on invoice images. 2. Invoice Input & Intelligence Layer 2.1 Invoice Capture • User can: o Click invoice photos using camera o Upload existing images or PDFs • Multi-page invoice support 2.2 OCR + Data Understanding • On-device OCR first, cloud fallback if confidence is low • Auto-detect and extract the following fields: Party & Document • Party (vendor) name • Party address • GST number • Invoice number • Invoice date • Due date • Purchase order reference (if present) Line Items • Item name • HSN (if present) • Quantity • Unit • Rate • Line amount Tax & Totals • CGST / SGST / IGST (if applicable) • Total tax • Invoice total Bank & Payment Details • Bank name • Account number • IFSC • UPI ID (if mentioned) People / Contacts • Owner name • Accounts contact • Phone number • Email ID Each extracted field must show: • Confidence indicator • Manual edit option • Ability to mark field as “verified” 3. Main Page (Home Dashboard) 3.1 View Selector (Primary Control) User can switch between: A. Party-Wise View Each row shows: • Party name (clickable) • Total outstanding amount • Current financial year purchases • Last invoice date • Nearest due date B. Invoice-Date-Wise View Each row shows: • Invoice date • Party name (clickable) • Invoice number • Invoice amount • Payment status (paid / unpaid / overdue) All rows and major data points are clickable. 4. Invoice Summary Page (Entry Point After Save) This is the first page after an invoice is captured and saved. Displays: • Party name (clickable) • Invoice number • Invoice date • Due date • Item list with quantity and price • Tax breakdown • Total amount Actions: • Make payment • Schedule payment • Set reminder on due date • Mark as paid / partially paid • Link to sales order(s) 5. Party Detail Page (Central Intelligence Page) Opened by clicking on any party name. 5.1 Basic Party Profile • Party name • Address • GST number • Notes (manually addable) 5.2 Past Orders Section Default view: • Last 3 invoices with: o Date o Amount o Payment status Expanded view (“View All”): • Current financial year total purchases • Date-wise invoice list • Item-wise aggregation • Amount-wise distribution 5.3 Bank Accounts Section • System auto-collects all bank accounts mentioned across: o Current invoice o Previous invoices of same party • Deduplicate accounts automatically • Display one account per card: o Bank name o Account number o IFSC / UPI Actions per account: • Link with UPI • Link with ICICI Bank • Set as default payment account 5.4 Owner / Contact Section For each person linked to the party: • Name • Role (Owner / Accounts / Sales / Other) • Phone number • Email ID User can: • Manually add contacts • Edit existing contacts With permission from phone: • Show last call date/time with that contact • Quick actions: o Call o Email o Message 5.5 Party Summary Section • Lifetime purchase value • Current year purchase value • Total paid • Total outstanding • Average payment delay • Upcoming dues timeline 6. Sales Order Linking & Reconciliation (New Section) 6.1 Sales Order Repository User can: • Manually create sales orders • Import sales orders (basic fields): o Sales order number o Customer name o Order date o Items o Quantity o Value 6.2 Linking Purchases to Sales Orders From Invoice Summary or Party Page: • Option: “Link to Sales Order” Linking rules: • One purchase invoice can be linked to: o One sales order o Multiple sales orders • Multiple purchase invoices can be linked to: o One sales order Linking logic: • Item-level mapping (preferred) • Amount-level mapping (fallback) • Partial quantity mapping supported 6.3 Sales Order View Each sales order shows: • Linked purchase invoices • Total purchase cost • Gross margin estimate • Pending purchases (if partially fulfilled) • Status: o Fully covered o Partially covered o Not covered 7. Payments & Reminders System • Pay using linked UPI or bank account • Schedule future payment • Auto reminders on due date • Payment status syncs across: o Invoice o Party page o Home dashboard o Sales order linkage 8. Technical & Architecture Requirements • Native iOS app (Swift / SwiftUI) • Modular architecture • Offline-first data storage with sync • Secure encryption for financial data • Permission-based access to: o Camera o Contacts o Call history (read-only metadata) • Robust deduplication engine: o Parties o Bank accounts
